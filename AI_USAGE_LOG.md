# AI 活用ログ — Team 05

審査項目「AI 活用度」の根拠資料。Day1〜Day3 を通して、AI ツールをどう使ったかを記録する。
**最低 1 日 3 件以上**の記入を目安に。

## 記入ルール

- 開発の節目ごとに追記する（5 分単位で書くより、機能/論点単位で書く）
- プロンプトは「実際に投げた文」を残す（要約は審査時に減点要因になりうる）
- 出力評価は **採用 / 一部採用 / 不採用** の 3 値で正直に
- 「採用しなかった理由」も書くと AI 活用の判断力アピールになる

### #001 ペルソナ具体化

- **時刻**：12:30
- **ツール**：ChatGPT
- **目的**：ハッカソンテーマに対して、誰のどのような課題を解くのかを明確にするため、初期ペルソナを具体化する。
- **プロンプト**：
  ```
  ## Who：都内の私立大学に通う大学4年生

  →就活がすでに終わり、夏休みに旅行に行ったが、現在夏休み明けで虚無っている状態
  ・最近はずっとNetflixを見ている
  ・高校までは部活で大学は緩めのサークル

  課題
  ・時間が余っているから新しい趣味を見つけたい
  ・お金をあまりかけたくはない
  ・今何やればいいかわからん

  今こんな感じでペルソナを設定して、Whoをふかぼっています。
  もうちょいありますかね？
  ```
- **出力サマリ**：
  - 「暇な大学4年生」ではなく、就活という外部目標が終わり、自由な時間はあるが熱量の向け先を失っている人として再定義した。
  - ペルソナ「ミサキ」を設定し、生活状況、心理、本人の発話、外的要因、行動、悩みを整理した。
  - 本質的には「暇」ではなく、「自分の好奇心が動き出す入口に出会えていない」ことが課題だと整理した。
- **評価**：採用
- **採用 / 不採用の理由**：以降の Who / What / How 設計の土台になったため採用。特に「自由な時間はあるが、何に熱量を向ければよいかわからない」という解釈が、プロダクトの課題設定に直結した。

### #002 Who, What, How の整理

- **時刻**：13:10
- **ツール**：ChatGPT
- **目的**：「熱狂とは何か」を定義し、Who / What / How の構造を整理する。
- **プロンプト**：
  ```
  # 熱狂とは？
  熱狂とは自分の根本にある好奇心に純粋に従うこと(生産性とか考えない、外的要因全無視)
  →外から与えられた予定ではなく、自分から次の行動を起こしたくなる状態。
  →最初からではなくて、やってたらだんだん湧き上がってくるもの(0→1, 1→10のプロダクト)
  (0→1だけでは熱狂しているといえないので、1→10も含める)

  # Who, What, Howの定義

  ## Who：都内の私立大学に通う大学4年生

  名前：ミサキ
  年齢：22歳
  属性：都内私立大学4年生
  状況：就活終了済み。卒業まで大きな予定がない。
  生活：週2でバイト、授業は少なめ。空き時間はNetflix、TikTok、Instagram。
  過去：高校までは部活で予定が埋まっていた。大学では緩めのサークルに入っていたが、今は活動も少ない。
  悩み：自由な時間はあるが、何かに夢中になれていない。
  制約：卒業旅行や遊びでお金も使いたいため、高額な趣味は始めづらい。
  本音：何か始めたいけど、何に興味があるのか自分でもよくわからない。

  今こんな感じの流れです
  ```
- **出力サマリ**：
  - 熱狂を「自分の根本にある好奇心に従い、自分から次の行動を起こしたくなる状態」と整理した。
  - 0→1 は「好奇心の入口を見つける」、1→10 は「小さく試した興味を熱狂に育てる」ことと定義した。
  - What として、「自分の好奇心の入口に出会えていない」「興味を行動に変換できていない」「外部目標がなくなり、自分から行動を選べない」ことを整理した。
- **評価**：採用
- **採用 / 不採用の理由**：「熱狂は最初から存在するものではなく、小さな行動を通じて育つ」という考え方が、以降のプロダクト案の軸になったため採用。

### #003 How のアイデア発散

- **時刻**：13:50
- **ツール**：ChatGPT
- **目的**：定義した課題に対して、どのようなプロダクトアプローチがあり得るかを発散する。
- **プロンプト**：
  ```
  - 興味から行動に変える動線が弱い
  →結果として時間はあることに虚無を感じてしまう
  - 自分の興味がないと思っている(=ニッチ)のものと触れる機会がない
  ・触れたことない領域
  ・知ってる領域の知らない項目

  こんな感じの課題としました。
  これを解決するためのプロダクト案をいくつか出してください
  ```
- **出力サマリ**：
  - 「偏愛ミッション」「となりの偏愛ガチャ」「ニッチ好奇心診断」「偏愛翻訳機」「虚無時間リプレイス」「偏愛借り物競争」「好き未満ログ」などの案を発散した。

- **採用 / 不採用の理由**：初期案としては有効だったが、「今日の小さな行動を提示する」だけでは単発で終わりやすく、継続性や実際の参加動機が弱いと判断したため一部採用に留めた。後の「ショート動画から体験会へ接続する」構造に発展した。
