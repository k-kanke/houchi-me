## プロダクト概要

> **誰の・どんな課題**: 就活後の大学生の「面白いのに行動に移せない（見て終わり）」「知らないニッチ体験に出会えない」「何にハマればいいかわからず受動的に時間を過ごしてしまう」という課題。

Curio Meet は、熱狂している人のショート動画を入口に、未知の趣味やニッチな体験と出会い、そのまま体験会に参加できるサービスです。
ユーザーはショート動画風フィードで気になる体験会を見つけ、詳細モーダルからその場で予約できます。
参加後にはログとポイントが残り、好奇心マップが成長することで、自分が何にハマりそうかを可視化できます。
一般ユーザーもクリエイターとして体験会を投稿でき、自分の好きなことを誰かの新しい趣味の入口にできます。

### 解決したい課題
1つ目は、SNSやショート動画で「面白そう」と思っても、実際の行動にはつながりにくいことです。既存のショート動画は興味を刺激しますが、体験予約や参加までの導線が弱く、結局「見て終わり」になりがちです。
2つ目は、自分では検索しないニッチな趣味や体験に出会いにくいことです。人はすでに知っている関心領域の中でしか探さないため、触れたことのない領域や、知っている領域の中の知らない項目に出会う機会が少なくなります。
3つ目は、就活や部活のような外部目標がなくなったとき、自由な時間を能動的に使うきっかけがないことです。時間はあるが、何に取り組めばよいかわからず、受動的なコンテンツ消費に流れてしまいます。

### ターゲットユーザー
メインターゲットは、就活が終わり、卒業までの自由な時間を持て余している都内の私立大学4年生です。
授業や予定が少なく、Netflix、TikTok、Instagramなどで時間を潰しているものの、「何かしたい」「何かにハマりたい」という気持ちはあります。
一方で、自分が何に興味を持てるのかわからず、いきなり習い事やコミュニティに参加するほどの熱量もありません。

また、サブターゲットとして、自分の好きな活動や偏愛を他人に共有したい一般ユーザー・クリエイターも想定しています。陶芸、ボードゲーム、茶道、古着リメイク、街歩き、フィルムカメラなど、自分の熱狂を初心者向けの体験会として投稿できます。

### コア機能
1. ショート動画風フィード

熱狂している人が投稿した体験会コンテンツを、TikTok / Reels のような縦型フィードで表示します。
ユーザーは受動的に眺めながら、自分では検索しない未知の体験と出会えます。

2. 体験会詳細モーダル・予約

気になる体験会を見つけたら、Home画面上のモーダルで詳細を確認できます。
日時、場所、参加費、定員、獲得ポイント、初心者歓迎、友達参加OKなどを確認し、そのまま予約できます。
独立した詳細ページに遷移させず、ショート動画から予約までの導線を短くすることが特徴です。

3. プロフィール・ポイント・好奇心マップ

予約・参加・参加後ログに応じてポイントが貯まり、称号や好奇心マップに反映されます。
好奇心マップでは、「手を動かす系」「空間を楽しむ系」「人と遊ぶ系」など、自分がどの領域に反応しているかを可視化します。

4. 体験会投稿

一般ユーザーもクリエイターとして、体験会を投稿できます。
タイトル、ジャンル、説明、画像、日時、場所、定員、参加費、初回無料設定、初心者歓迎設定などを入力し、自分の好きな活動を体験会として公開できます。

5. 参加後ログ

体験会に参加した後、感想、面白かった度、またやりたい度、次に気になるジャンルを記録できます。
ログ保存によってポイントが加算され、好奇心マップも更新されます。

## セットアップ手順

```bash
# Frontend
cd frontend
npm install
npm run dev
```

### Docker で起動する

`frontend/.env.example` をコピーして `frontend/.env` を作り、Supabase の URL / anon key を設定する。

```bash
cd frontend
cp .env.example .env
docker compose up --build
```

アプリは `http://localhost:5173` で開く。
開発用コンテナは起動時に `npm ci` を実行して、`node_modules` ボリュームの依存ずれを避ける。

本番相当の静的配信を確認したい場合:

```bash
cd frontend
docker compose --profile prod up --build web-prod
```

こちらは `http://localhost:8080` で開く。
`web-prod` は `.env` の `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` を build 時に埋め込む。
